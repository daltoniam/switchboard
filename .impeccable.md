# Design Context

## Target Audience
Solo developers and small engineering teams configuring MCP integrations. Technical users comfortable with CLIs and config files who prefer a quick visual overview over dense documentation.

## Use Cases
- Configuring integration credentials (primary)
- Checking which integrations are healthy/connected
- Monitoring tool usage and performance metrics
- Quick-glancing at system status

## Brand Personality / Tone
Engineering tool — clean, functional, utilitarian. Prioritize usability and information density over decoration. A small amount of flair (subtle color accents, crisp type hierarchy) but never at the expense of clarity. Think: well-designed CLI output rendered in a browser, not a marketing dashboard.

## Constraints
- Go templ templates (server-rendered HTML, inline CSS, no build step)
- No external CSS/JS frameworks — all styles inline in base.templ
- Must work on modern browsers, responsive down to 768px
- No heavy fonts — system font stack preferred for speed
